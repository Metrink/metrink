1) Start Pushing Metrics into Metrink

Download the Metrink Agent: Linux or Windows(beta)

The Metrink agent uses the RESTful API to send standard CPU, memory, disk, and network metrics to Metrink. With the Metrink agent, you can easily add additional scripts to be run and collect metrics. So expanding the metrics you collect has never been easier. Learn more about the Metrink Agent.

RESTful HTTP API

Metrics can be sent directly to Metrink's RESTful HTTP API from any application. This is most applicable to developers who want to push application specific metrics directly from their code. By combining both application performance metrics and device metrics, you can get the complete view of your infrastructure in a single graph. For additional information, see the API specification.

2) Ask Questions of Your Metrics

After a few minutes of gathering metrics, you can begin graphing! Metrink's Query Language provides a powerful method for asking questions of your metrics and quickly getting answers. Use the wrench icon to guide you in constructing queries.

All of the information for a query is contained in the URL, so by simply copy and pasting the URL others can see the same graph. You can also easily save any graph to a dashboard.

Sample Queries
  • -30m m("*", "cpu", "load") Load across all devices for the last 30 minutes
  • -30m m("*", "cpu", "load", -1d) Identical to above, but with yesterdays data as well
  • -30m m("*", "cpu", "load") >| avg Add a horizontal line displaying the average
  • 2013-01-01 7:00 to 2013-01-03 7:00pm m("*", "cpu", "load") Absolute time
  • 7:00 m("*", "cpu", "load") Absolute time from 7am today to now
  • -1h m("*", "cpu", ["idle", "user"]) Multiple metrics
  • -1h m("web", "cpu", "idle") + m("db", "cpu", "idle") Add two metrics together
  • -1h m(["web", "db"], "cpu", "idle") | sum Different way to graph the same metrics
  • Read more about the query language.

3) Configure Some Alerts

Leveraging the query language you can easily construct alerts. With alerts Metrink is constantly watching your infrastructure. Alerts are stateful to help prevent false alerts. You can either email or page someone to respond to an alert.

To learn more, take a look at the Metrink's Alerting Language.

Sample Alerts
  • m("web", "cpu", "load") > 0.5 do email-admin When load on the web box is above 0.5, email the administrator.
  • m("*", "cpu", "load") > 0.5 for 5m do email-admin When load on the web box is above 0.5 for 5 minutes, email the administrator.
  • m("*", "cpu", "load") > 0.5 for 5m do email-admin When load across any device is above 0.5 for 5 minutes, email the administrator.
  • m("web", "cpu", "load") > 0.5 do page-admin When load on the web box is above 0.5, page the administrator.
  • Read more about actions and alerts.
